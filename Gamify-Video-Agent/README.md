# Project Gamify-Video-Agent 🎮
### Multimodal Temporal Reasoning for Interactive Procedural Learning

---

## 🎯 Purpose of The Project
Traditional video tutorials are passive learning mediums with low retention rates. **Gamify-Video-Agent** is an autonomous system that transforms static instructional videos (e.g., industrial assembly, medical procedures, or software walkthroughs) into active, interactive simulations. 

By leveraging Large Vision Models (LVMs) and Agentic code generation, the system "watches" a video, understands the procedural steps, identifies spatial "hotspots," and programmatically generates a playable React interface. Users must then point, click, or sequence actions over the video to prove they have mastered the task.

---

## 🧠 Foundational Concepts & Enterprise Relevance
This project focuses on the frontier of **Multimodal AI Engineering**:

* **Temporal Multimodal Understanding:** Moving beyond static captions to understand *when* and *where* actions occur in a 3D space over time (Temporal Grounding).
* **Vision-to-Code Synthesis:** Using AI Agents to translate visual observations (pixels) into functional software components (React/Javascript event listeners).
* **Action Recognition & State Tracking:** Tracking the state of a physical system (e.g., "The safety pin is now removed") purely through visual frames.
* **Low-Latency Edge Vision:** Deploying Small Vision Models (SVMs) locally to perform real-time analysis, ensuring data privacy and offline capability—a critical requirement for enterprise "on-the-job" training.

## 🛠 Technical Requirements

### Phase 1: Visual Ingestion & Scene Decomposition
* **Activity Segmenting:** Implement an agent that breaks a video into "Logical Steps" based on visual cues and timestamps.
* **Spatial Hotspot Mapping:** For every instructional step, the agent must identify the exact $(x, y)$ coordinates of the objects being interacted with.
* **State Machine Extraction:** The agent must generate a JSON-based state machine describing the workflow (e.g., `Step 1: Locate Tool A` -> `Step 2: Apply Tool A to Point B`).

### Phase 2: Agentic App Synthesis (The "Gamifier")
* **Dynamic Overlay Generation:** Develop an agent that generates a transparent React-based interactive layer that sits on top of the video player.
* **Game Mechanics:** Implement three levels of interaction:
    * **Level 1 (Guided):** The user is prompted to click a highlighted area to continue the video.
    * **Level 2 (Sequencing):** The video pauses, and the user must click three objects in the correct order shown.
    * **Level 3 (Audit):** The user must identify a "Safety Error" or a skipped step in a modified version of the video.

### Phase 3: Deployment & Scoring
* **Local Inference:** The Vision-Language Model (e.g., LLaVA, Moondream, or InternVL) must run on the local desktop or a private cloud instance.
* **Leaderboard Engine:** Implement a backend to track user accuracy, "Time to Action," and successful completion rates.

## 📜 Rules of Engagement (First-Principles Learning)

1.  **Zero-Framework UI:** You are **prohibited** from using pre-built interactive video platforms (e.g., H5P, Adobe Captivate). The interaction logic and overlay positioning must be generated by your Agent in raw React/CSS.
2.  **Autonomous Analysis:** Interns cannot provide manual labels for the video. The Agent must look at the pixels and determine where the "hotspots" are without human assistance.
3.  **Hardware Constraint:** Frame analysis must be optimized to run on consumer-grade hardware (e.g., 8GB-12GB VRAM) using quantization.
4.  **IDE Specificity:** The project must be developed entirely using **Cursor** or **Antigravity**, leveraging AI-pair programming to handle the complex state-management between video timestamps and UI overlays.

## 📊 Evaluation & Grading Criteria

| Metric | Goal | Description |
| :--- | :--- | :--- |
| **Spatial Precision** | < 15px Drift | Accuracy of the generated clickable hotspot relative to the actual object in the video. |
| **Temporal Sync** | < 300ms | The interactive prompt must appear and disappear in perfect sync with the video frames. |
| **Code Generation** | 100% Valid | The Agent-generated React code must build and run without manual "bug-fixing." |
| **User Retention** | > 80% Mastery | Effectiveness of the gamified version vs. a control group just watching the video. |
| **Inference Speed** | > 5 FPS | The speed at which the local agent can process and segment new video input. |
